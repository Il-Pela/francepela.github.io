Manual installation steps for previous versions of WSL 2

Dec 9, 2022

Sources:

  • [windows docs](Manual installation steps for previous versions of WSL | Microsoft Learn)

Step 1 - Enable the Windows Subsystem for Linux

Before installing any Linux distribution on Windows, you must enable the optional “Windows Subsystem for Linux” feature. Open PowerShell as administrator (Start menu > PowerShell > right-click > Run as administrator) and enter this command:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

We recommend that you proceed to step 2, Upgrade to WSL 2, but if you want to install only WSL 1, you can now restart your computer and go to [[#[](https://learn.microsoft.com/it-it/windows/wsl/install-manual step-6—install-your-linux-distribution-of-choice)Step 6 - Install your preferred Linux distribution|Step 6 - Install your preferred Linux distribution]]. To upgrade to WSL 2, wait for your computer to restart and proceed to the next step.

Step 2 - Verify the requirements for running WSL 2

To upgrade to WSL 2, you must be running Windows 10…

  • For x64 systems: Version 1903 or later, with build 18362 or later.
  • For ARM64 systems: version 2004 or later, with build 19041 or later.

or Windows 11.

[!INFO] Builds lower than version 18362 do not support WSL 2. To update your version of Windows, go to the Windows Upgrade Assistant.

To check the build version and number, press the Windows key + R, type winver, and select OK. Update to the latest version of Windows in the Settings menu.

[!INFO] If you are running Windows 10 version 1903 or 1909, open “Settings” from the Windows menu, go to “Update & Security” and select “Check for Updates”. The build number must be 18362.1049+ or 18363.1049+, with the secondary build greater than .1049. More information: Support for WSL 2 is available in Windows 10 versions 1903 and 1909.

Step 3 - Enable virtual machine features

Before installing WSL 2, you must enable the optional Virtual Machine Platform feature. Your computer will require virtualization functionality to use this feature.

Open PowerShell as administrator and run:

PowerShellCopy

dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

To complete the WSL installation and upgrade to WSL 2, restart your computer.

Step 4 - Download the Linux Kernel Update Package

  1. Download the latest package:

    • WSL 2 Linux Kernel Update Package for x64 Computers

[!NOTE] If you are using an ARM64 computer, download the ARM64 package instead. If you don’t know exactly what type of computer you’re using, open Command Prompt or PowerShell and type: . Warning: Non-English versions of Windows may need to change the search text, translating the string “System Type”. You may also need to escape quotation marks for the find command. For example, in German .systeminfo | find "System Type"``systeminfo | find '"Systemtyp"'

  1. Run the update package that you downloaded in the previous step. Double-click to run: You will be prompted to set elevated privileges. Select Yes to approve this installation.

Once the installation is complete, proceed to the next step, which is setting WSL 2 as the default version when installing new Linux distributions. Skip this step if you want new Linux installations to be set to WSL 1.

[!INFO] For more information, read the article about changes to the WSL2 Linux kernel update, available on the Windows command line blog.

Step 5 - Set WSL 2 as the default version

Open PowerShell and run this command to set WSL 2 as the default version when installing a new Linux distribution:

PowerShellCopy

wsl --set-default-version 2

Step 6 - Install your preferred Linux distribution

  1. Open Microsoft Store and choose your favorite Linux distributions.
    The following links open the Microsoft Store page for each distribution:

    • Ubuntu 18.04 LTS
    • Ubuntu 20.04 LTS
    • Ubuntu 22.04 LTS

  2. On the distribution page, click “Get.”

The first time you start a newly installed Linux distribution, a console window will open and prompt you to wait a few minutes for the files to be unpacked and stored on your PC. All subsequent starts should take less than a second.

You must then create a user account and password for your new Linux distribution.
